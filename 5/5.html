Calculator: Part 2

Now that you have created your first two React Components, the next step is to build a custom function in order to keep the data for the keys on the keypad separate.

Create Keys File

First step is to create a new folder inside of the src folder and call it utils. Once you have created the new folder, create a new file inside with the name useKeys.js. This is where we are going to write the code that will return the data needed for the keyboard keys.

Open up the file and put the following code inside.

const useKeys = () => {
  const keys = [
    {value: "C", color: "#3d4759", width: 3},
    {value: "(", color: "#3d4759", width: 3},
    {value: ")", color: "#3d4759", width: 3},
    {value: "/", color: "#ff9f24", width: 3},
    {value: "7", color: "#7e7e7f", width: 3},
    {value: "8", color: "#7e7e7f", width: 3},
    {value: "9", color: "#7e7e7f", width: 3},
    {value: "*", color: "#ff9f24", width: 3},
    {value: "4", color: "#7e7e7f", width: 3},
    {value: "5", color: "#7e7e7f", width: 3},
    {value: "6", color: "#7e7e7f", width: 3},
    {value: "-", color: "#ff9f24", width: 3},
    {value: "1", color: "#7e7e7f", width: 3},
    {value: "2", color: "#7e7e7f", width: 3},
    {value: "3", color: "#7e7e7f", width: 3},
    {value: "+", color: "#ff9f24", width: 3},
    {value: "AC", color: "#eb4034", width: 3},
    {value: "0", color: "#7e7e7f", width: 3},
    {value: ".", color: "#7e7e7f", width: 3},
    {value: "=", color: "#ff9f24", width: 3}
  ];
  return keys;
};

export default useKeys;

As you can see there really is nothing special happening here, but it's the first step in separating out re-useable data to another file. We don't need to import React here either because we are just simply writing plain old JavaScript.

We just have the data for all the keys of the calculator that we will later map out into components. Moving it out into a separate file means we can keep the files we choose to use it clean and less convoluted with data. We can then just focus on what we want the user to see in the Key component file.

Create the Key component

This now leads us nicely onto the next thing we are going to need to create, which is the Key component that will form the display of each of the individual Keys. So what is going to be the best way to approach this? Are we going to write the same thing for each of the keys or is there a better approach?

Okay, ready to move on?

Let's go ahead and create a new file inside of the components folder called Key.js. This file is going to contain the code that will style each of our buttons.

Go ahead and place the following code in the file.

import React from "react";

const Key = ({ data }) => {
  return (
    <>
      <div
        style={{
          width: `calc((100% / 12) * ${data.width})`,
          backgroundColor: data.color,
          border: "1px solid black",
          boxSizing: "border-box",
          cursor: "pointer",
          padding: "0"
        }}
      >
        <button
          style={{
            border: "none",
            backgroundColor: "transparent",
            width: "100%"
          }}
        >
          <h2 style={{ textAlign: "center", color: "white" }}>{data.value}</h2>
        </button>
      </div>
    </>
  );
};

export default Key;

If you look closely at the parameter being passed into the Key function, we are destructuring the parameter. In the world of React, this parameter is referred to as props. Any data that we pass down the component tree is called props. Props are always objects, so going back to what you learnt earlier, you can now destructure the props to just pull the data straight out.

If you look closely at the code, the data that is being passed down is the same as the data for each of the keys. Coincidence? I think not.

Next, we are going to map out the keys onto the Keypad. We need to map the reusable Key component in the Keypad component.

Mapping the Keys

Now head on into the Keypad.js file; we are going to fill in the code that we place in there earlier.

We now need to use the useKeys file that was created earlier to bring the data for the keys into the Keypad component. At the top of the file go ahead and import the function to use.

import useKeys from "../utils/useKeys";

Once you have done that, you now have access to the useKeys function to use inside of the Keypad component. Right before the return statement inside the functional component, add the following line, which will assign the value being returned from the useKeys function into a variable.

const keys = useKeys();

This naming of useKeys should help you to feel more comfortable with the naming convention of React Hooks and hopefully make you feel more at ease with writing them. On the surface hooks are just functions, only there is some React magic happening under the hood to make your life easier.

When you have done that, now you can use the keys which will be an array of all the objects that define the keys data.

The next step is to import the Key component into the Keypad component and then map out the keys. Then, where we left the empty code earlier we can now fill in with the following code, which will map out the keys.

{keys.map((key) => {
  return <Key data={key} />
})}

Finally, we are going to import the Keypad and CalculatorDisplay components into our App.js in order to render both components so they appear on your web page. Take a close look at the App.js file and see if you can figure out how to display both these components in your web page. The answer is in the next lesson, but try it out for yourself first.

If you have done everything as you should, the outcome should be something that resembles a calculator if you go and look at the web page running on your localhost.

Great! So you now have something that resembles a calculator but at this point it still doesn't do anything. The next step is going to be adding in the functionality to the calculator. This is going to require passing props around from the app to the child components.
