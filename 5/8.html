React Tic Tac Toe

Let's build Tic Tac Toe again, but this time in React.

This is a very important step to realize the advantage React has over no-framework-DOM-manipulation, and help you to think in React.

Step 1: Build a static version in React

Use create-react-app to generate a new app for Tic Tac Toe.

Ignore state for now. Focus on the return() in your function and your CSS.

You may also try out styled-components!

Step 2: Identify the State

To make your app interactive, you will need to trigger changes through state.

React state stores a set of data that needs to be changed when user interacts with your app. And bear in mind a state change will trigger a rendering, therefore updating what the user sees.

For example, if you want a turn variable to keep track of number of turns, then you need to put that into your initial state.

For Tic Tac Toe, it is recommended that you start with the following state:

const [board, setBoard] = useState([
      '', '', '',
      '', '', '', 
      '', '', ''
    ])

In your return(), you can display each box like so:

return (
  <div>
    {
      board.map((value, index) => {
        return (
          <div key={index} id={index}>{value}</div>
        )
      })
    }
  </div>
)
  1. board is an array of 9 strings ('', 'X' or 'O')
  2. Using .map, we transform each element of the array into <div>
  3. The JSX elements you generated using .map must have key prop. The key for each element must be unique, in this case we index to achieve that.

Step 3: Identify the events

What are the events user can do to interact with your app? Define the event handler and attach it to the right DOM.

Use React Dev Tool (the 'React' tab inside Chrome Developer Tool) to see live changes made to the state

const handleClick = e => {
  console.log(e.target.name) // will print the 'name' attribute of the button you press
}

return (
  <div>
    <button name="a" onClick={handleClick}>A</button>
    <button name="b" onClick={handleClick}>B</button>
  </div>
)

Step 4: Test and Improve!

Test out edge cases:

  • pressing the same square multiple times
  • test out a tie game
  • how does the game restart
  • etc etc

Compare your working React Tic Tac Toe with the old one.
