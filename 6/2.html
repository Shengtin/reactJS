Nextagram API Endpoints

React JS only handles the frontend - the UIUX.

We can create a login form with React, but pressing the 'Login' button will not do anything unless we instruct it to interact with a backend server.

Therefore, for Nextagram, we have provided you a backend server built with Python.

Your React app will 'talk' to the server through a few API endpoints that we have prepared for you.

Essentially, to 'talk' to a server, we make API calls (HTTP requests) to the API endpoints. We can do that through Fetch, which is a built-in feature in most of the modern browsers.

For our React app, we will use a library called Axios to make the API call process easier, the call looks very similar to Fetch, but less wordy.

axios({
  method: 'POST',
  url: '<API Endpoint URL>',
  data: {
    hello: 'world',
  }
})
.then(data => {
  // The server responds to us with data
})
.catch(error => {
  // handle error here
})

Nextagram API

The documentation for Nextagram API are located here:
https://documenter.getpostman.com/view/2792518/RzZ6HLBy

Let's experiment! Visit one of the endpoints on your browser, try this:
https://insta.nextacademy.com/api/v1/users/

If the API endpoints take a while to load don't worry! We are hosting this API app on Heroku's free tier which turns the server off when there are no web requests. Subsequent requests will get faster.

The result you see is a list of all users registered to Nextagram. This user list data is in JSON format.

What you just did was that you tapped into a server and retrieved information from it by making a GET HTTP request.

However, if you try visiting this: https://insta.nextacademy.com/api/v1/login, you get Method Not Allowed error. That is because according to the documentation, Login endpoint has to be a POST request. By pasting a URL on your browser, you are actually making a GET request.


You may visit https://insta.nextacademy.com to check out the Nextagram app built using Python + Flask (without React). Our API endpoints in this react course is actually communicating with that same server, which means the data you made through the API will be reflected on the website.
