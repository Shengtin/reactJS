What is React?

React is often described as ‚Äúthe V in the MVC structure‚Äù. It is a JavaScript library for building user interfaces. The big idea behind React is this: what if you could create your own HTML element that has customized functionality? This is known as a component.

React Component

At the heart of all React applications are components. A component is a mixture of HTML and JavaScript that captures all of the logic required to display a small section of a larger UI.

For example, here's a React component named GreetMessage:

const GreetMessage = props => {
  return <h1 className="text-white">Hello, {props.name}!</h1>;
}

We can reuse multiple GreetMessage components in other components, for example:

const Welcome = () => {
  return (
    <div id="container">
      <GreetMessage name="Harry" />
      <GreetMessage name="Ron" />
    </div>
  )
}

1. JSX

So what's this weird combination of JS + HTML hybrid that you're seeing from above example?
The syntax is known as JSX, where you write HTML-like Javascript. Yeah, those "HTML" tags are actually still JavaScript!

2. className instead of class

Since the HTML-like syntax is actually JS, and class is a reserved keyword in JS, Facebook had to call it something else - className

Take note that certain HTML attributes with dashes like tab-index will follow JS camelCase convention and should be written as tabIndex.

3. Calling variables in JSX using {}

In vanilla JS, if you want to use JS to add name to the following span:

<h1>Hello, <span id="name"></span>!<h1>

You will have to do something like this:

document.getElementById('name').innerHTML = guestName

In React, you simply write:

<h1>Hello, {guestName}!</h1>
