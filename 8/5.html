Folder Structure

By now, you already have at 4 different components in separate JS files: App.js, HomePage.js, Loader.js (or whatever you named it) and UserImages.js.

In next challenge, you will be introduced to React Router where you will have even more components that represent Pages.

As you can see, it's best we start organizing these files into their own folders.

src/pages

This folder is meant for each page of your app. React Router will redirect users from one page to another, and each page will then subsequently load more components.

Create a folder called pages, and move your HomePage.js into this folder.

src/containers

We store Smart Containers here. They are the non-pages components that contain logic (states, api calls, etc).

Move your UserImages.js here.

src/components

We store Dumb Components here - those presentational components that can be reused for UI purpose (buttons, navbar).

Move your Loader.js here.

Your folder structure should look like this now:

src
├── components
|   └── Loader.js
├── containers
|   └── UserImages.js
├── pages
|   └── HomePage.js
├── App.js
...
...

Note: Don't forget to update the import of these moved components in App.js!
