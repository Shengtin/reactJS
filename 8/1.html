React Props

Unlike state which only lives inside a component, prop is the value that was passed down by parent.

Take a look at the following example:

Parent Component: <App/>

import React, {useState} from 'react'
import CommentBox from './CommentBox'

function App () {
  // we can also define state like this without using constructor
  const [title, setTitle] = useState("Hello World")


  return (
    <section>
      <h1>Welcome</h1>
      <CommentBox heading={title} />
    </section>
  )
}

export default App
  1. App is the parent of CommentBox
  2. CommentBox will receive a prop (property) called "heading"
  3. The value of this heading prop is 'Hello World'

Child Component: <CommentBox/>

import React from 'react'

const CommentBox = props => {
  return (
    <div>
      <label>{props.heading}</label>
      <textarea cols="30" rows="10" />
    </div>
  )
}

export default CommentBox
  1. To access props using a functional component, we expose props as function argument, then access props.heading.

You can go 1 step further and destructure the props to get the heading without having to call props.heading.

Props is ultimately just an object so we can destructure objects to get the values that we need.

import React from 'react'

const CommentBox = ({heading}) => {
  return (
    <div>
      <label>{heading}</label>
      <textarea cols="30" rows="10" />
    </div>
  )
}

export default CommentBox
