Reuseable Functional Component

When you think back to basics of just HTML, CSS and JavaScript, one of the difficulties faced is having to write the same piece of code over and over again.

For example, if you want to have a Navbar on all 5 of your pages, you have to write that same Navbar code 5 times.

In comes React. It is possible to write reuseable components using React, but that also doesn't mean that each time we use it, it has to be exactly the same.

With the use of props you can create components that depend on the props being passed in to define how they look or work.

Let's use the LoadingIndicator to demonstrate how you can do that.

const LoadingIndicator = () => {
  return (
    <svg
      xmlns="http://www.w3.org/2000/svg"
      xmlnsXlink="http://www.w3.org/1999/xlink"
      style={{
        margin: "auto",
        background: "transparent",
        display: "block",
        shapeRendering: "auto"
      }}
      width="200px"
      height="200px"
      viewBox="0 0 100 100"
      preserveAspectRatio="xMidYMid"
    >
      <circle
        cx="50"
        cy="50"
        r="32"
        strokeWidth="8"
        stroke="#fe718d"
        strokeDasharray="50.26548245743669 50.26548245743669"
        fill="none"
        strokeLinecap="round"
        transform="rotate(182.103 50 50)"
      >
        <animateTransform
          attributeName="transform"
          type="rotate"
          repeatCount="indefinite"
          dur="1s"
          keyTimes="0;1"
          values="0 50 50;360 50 50"
        ></animateTransform>
      </circle>
    </svg>
  );
};

Why?

React was created to make it so that developers could write less code and speed up the development time.

Reuseable components are one of the key factors in achieving this. By building reuseable components we can rely on just changing the props to give a different output, rather than having to write the same component out all over again with different values.

In the example of the loading indicator above the values are still being hard-coded in, so no we want to make the values dynamic. We can achieve this by incorporating props that will provide some of the values to change the appearance.

const LoadingIndicator = ({width, height, color}) => {
  return (
    <svg
      xmlns="http://www.w3.org/2000/svg"
      xmlnsXlink="http://www.w3.org/1999/xlink"
      style={{
        margin: "auto",
        background: "transparent",
        display: "block",
        shapeRendering: "auto"
      }}
      width={width}
      height={height}
      viewBox="0 0 100 100"
      preserveAspectRatio="xMidYMid"
    >
      <circle
        cx="50"
        cy="50"
        r="32"
        strokeWidth="8"
        stroke={color}
        strokeDasharray="50.26548245743669 50.26548245743669"
        fill="none"
        strokeLinecap="round"
        transform="rotate(182.103 50 50)"
      >
        <animateTransform
          attributeName="transform"
          type="rotate"
          repeatCount="indefinite"
          dur="1s"
          keyTimes="0;1"
          values="0 50 50;360 50 50"
        ></animateTransform>
      </circle>
    </svg>

Remember that you are writing JSX and not just HTML, so your attributes on your HTML need to follow the JSX format so that you do not face any errors in your code.

By taking in a width, height and color props you can change how the loader will look on different pages.

<LoadingIndicator width="300px" height="300px" color="green" />

<LoadingIndicator width="500px" height="500px" color="blue" />

Both are using the exact same component but the outcome is entirely different. Different size and different color.
