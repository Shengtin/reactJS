Object

Take a look at the following car Javascript object:

const owner = 'nicholas'

const car = {
  brand: 'Tesla',
  model: 'Model X',
  [owner]: {
    age: 21,
    gender: 'male',
  },
  drive: function() {
    console.log('psssttttt')
  }
}

console.log(car.brand) // Tesla
console.log(car['model']) // Model X
console.log(car.nicholas) // {age: 21, gender: 'male'}

// These 2 are equivalent:
car.drive() // psssttttt
car['drive']() // psssttttt

Inside the car object, brand, model and drive are known as property name or property key, while their respective values such as 'Tesla' or 'Model X' are known as property value.


1. Computed Property

The use of square bracket around a property name is known as computed property.

When we write

const owner = 'nicholas'
{[owner]: {age: 21, gender: 'male'}}

Due to the square brackets, the key [owner] will refer to the variable owner.

Since const owner = 'nicholas', it becomes:

{nicholas: {age: 21, gender: 'male'}}

2. Accessing object properties

To access properties on an object, you can either use dot notation: car.brand, or square bracket notation: car['brand'].


3. Function as property value

We can also put function as an object property value. For example:

const car = {
  ownerName: 'Nicholas',
  drive: function() {
    console.log(`${this.ownerName} drives his car away...`)
  }
}

It is also completely equivalent to:

const car = {
  ownerName: 'Nicholas',
  drive() {
    console.log(`${this.ownerName} drives his car away...`)
  }
}

The keyword this exists whenever a function is defined.

this by default will refer to the owner of the function. In this case, the car object owns the function drive().

Inside the function drive(), ownerName is undefined to the function. Therefore, we use this.ownerName to access "car's ownerName".


4. Shorthand Property Names

Basically, this:

const price = '$30,000'
const car = {
  price: price,
  model: 'Tesla'
}

can be rewritten as:

const price = '$30,000'
const car = {
  price, //shorthand here
  model: 'Tesla'
}
