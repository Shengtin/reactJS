Brief History of React

Programming languages are rapidly evolving and always changing. This is just the nature of the Software Development world.

React is no different. Things have changed over the years as the React development team continues to improve the framework and features to make it easier to use, faster and more efficient.

React was designed to cut down development time for frontend applications.

What was it like at the start?

React was first introduced back in 2011 and things were a lot different back then.

When React was first released, JavaScript was still in ES5 so there were no Arrow Functions, Spread Operators or Destructuring, amongst a host of other changes.

In the beginning React components were cumbersome to build and less readable than they are today.

Below is an example of how React components were created once upon a time not so long ago.

var createReactClass = require('create-react-class')
var App = createReactClass({
  render: function() {
    return <h1>Hello World</h1>
  }
})

This would be a very simple Hello World app before the introduction of ES6.

getInitialState: function() {
  return {count: 0}
}

As you can see, everything is treated as JavaScript objects and the component is being structured around the keys and values provided.

In comes ES6

With the introduction of ECMAScript 2015 (ES6) in 2015, it brought with it a whole host of changes. We are not going to go through them but we will see the changes it allowed for React.

Let's see how the same component above would have been written with the introduction of ES6.

import React from 'react'

class App extends React.Component {
  constructor(props){
    super(props);
    this.state = {
      count: 0
    }
  }

  render(){
    return <h1>Hello World! Count: {this.state.count}</h1>
  }
}

As you can see it completely changes the look and feel of React. It's cleaner, much easier to read and development time had improved dramatically.

However, functional components at this time could not contain state which meant they were purely for display purposes and would rely on props being passed from the parent to function.

What's next?

As you go through this course you will see that React has once more advanced quite dramatically around mid 2019 with the introduction of Hooks, and thankfully, with these changes, came huge benefits that make React much simpler to work with and make it much more beginner friendly.
