CRA Folder Structure

Open my-app folder in new VSCode window

When working with React code, you will deal with multiple files and folders that grow large over time. Avoid mixing non-related code files under the same VSCode window/workspace.

Open up the whole my-app folder create-react-app has generated for you in a new vscode window, take a look at the sidebar, you should be looking at a list of files and folders.

Folder Structure of CRA (create-react-app)

On top level, you will see public and src folders, and several files such as package.json. Let's take a look at a few important ones.

index.html and index.js

Open up public/index.html, which is basically the only real HTML in your entire React app. Notice how there is only a single div:

<div id="root"></div>

Tips: You can use cmd + p (Mac) or ctrl + p (Windows) to open a dialog which allows you to quickly search for a file to open

Next, take a look at src/index.js:

ReactDOM.render(<App />, document.getElementById('root'));

React will take the <App /> component and throw it into the only <div> tag, but what exactly is the <App /> component? Open up App.js

App.js

This the entry point of your app. You will write React code here. Try changing the HTML and see changes get applied on localhost:3000

package.json

What is package.json?

This file list out all external libraries that you have installed for app. The code for all these external libraries are located in node_modules folder.

React does not dictate how you should structure your files and folders. As your react app grows larger, you will need some kind of system to organize the JS files. You will learn more about this later.

This is React's official response to handling file structure: https://reactjs.org/docs/faq-structure.html
