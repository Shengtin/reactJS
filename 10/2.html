Pair Programming React Todo List

Let's take a break from Nextagram and build a todo list to strengthen your React Thinking:

You are going to build a very simple app like this (Don't bother with the code, reading/using it will just overcomplicate your progress)

Generate a new app using create-react-app, and build a todo list with the following requirements:

App Structure

You will have 3 Components:

1. <App/>

Initialize it with the following state:

  const [todos, setTodos] = useState(
    [
      { id: 1, task: 'Walk the dog', done: false },
      { id: 2, task: 'Water the flower', done: true}
    ]
  )

The todos state (which is an array of objects), will live in <App /> parent component, this is because both <TodoItem/> needs to read todos state, and <TodoForm/> component needs to execute setTodos function upon submission.

2. <TodoItem/>

This is the component for each todo list item.

It should be a presentational component without state. It simply receive todos from parent as props, and render them nicely.

The "delete button" with its onClick event should also live here.

3. <TodoForm/>

This component will render a form that consist of a Text Input and Submit Button.

You will have a state called text to store the characters users type into the form input:

const [text, setText] = useState('')

const handleSubmit = e => {
  e.preventDefault()
}

const handleInput = e => {
  setText(e.target.value)
}

return (
  <form onSubmit={handleSubmit}>
    <input onChange={handleInput} value={text} type="text" />
    <input type="submit" value="Add Task" />
  </form>
)

App Behavior

  1. Users can use the form to add new task.
  2. Users can delete a task by pressing a button next to it.
  3. Users can mark a task as done by clicking on the task itself.
  4. There should be obvious styling that differentiate between completed and incomplete tasks.

Better UIUX

  1. Form's text input should get cleared after adding a task
  2. Do not allow users to add task with empty string
  3. Make it look awesome!

Important

Be very careful NOT TO mutate states directly:

// do not do these:
todos.push(...)
todos[0] = ...

Rather, try doing this:

To add new items into an array state:

setTodos([
  ...todos,
  {id: 3, task: 'new task'}
])
